## Frequently Asked Questions

### Is GreenKube free?

Yes! GreenKube is **100% open source** under the Apache 2.0 license. It's free to use, modify, and distribute.

### Do I need an Electricity Maps token?

No. GreenKube works without an Electricity Maps token by using a default carbon intensity value (500 gCOâ‚‚e/kWh). For more accurate, region-specific data, we recommend getting a free token at [electricitymaps.com](https://www.electricitymaps.com/).

### Which Kubernetes distributions are supported?

GreenKube works with any standard Kubernetes distribution, including:
- Amazon EKS
- Google GKE
- Azure AKS
- Minikube
- k3s
- OpenShift
- Self-managed clusters

### Can I use GreenKube without OpenCost?

Yes. OpenCost integration is optional. Without it, cost data will not be available, but energy and carbon metrics will still work.

### How much resources does GreenKube consume?

With default settings, GreenKube requests 100m CPU and 128Mi memory, with limits of 500m CPU and 512Mi memory. The actual usage is typically well below the limits.

### Can I contribute?

Absolutely! Check out our [Contributing Guide](/contributing/) to get started. We welcome code, documentation, bug reports, and feature ideas.
